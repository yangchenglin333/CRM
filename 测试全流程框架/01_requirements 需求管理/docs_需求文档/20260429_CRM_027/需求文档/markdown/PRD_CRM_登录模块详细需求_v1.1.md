# CRM系统 - 用户登录模块详细需求文档

## 1. 需求概述

### 1.1. 功能定位
用户登录模块是CRM系统的入口，负责身份验证和会话管理，确保系统安全性和用户体验。

### 1.2. 需求来源
根据业务/产品描述：
- 用户登录：支持正常登录、用户名密码为空、用户名密码错误等场景
- 边界值校验：用户名4-50字符、密码6-32字符
- 格式校验：用户名仅支持中文、英文、数字、下划线
- 安全机制：连续5次失败锁定账号，10分钟自动解锁

## 4. 业务规则

### 4.1. 用户名规则

```
用户名格式：^[\u4e00-\u9fa5a-zA-Z0-9_]+$
说明：
- 支持中文（\u4e00-\u9fa5）
- 支持英文（a-zA-Z）
- 支持数字（0-9）
- 支持下划线（_）
- 不支持其他特殊字符
```

### 4.2. 密码规则

```
密码格式：任意字符
说明：
- 长度：6-32字符
- 建议包含大小写字母、数字、特殊字符（非强制）
```

### 4.3. 锁定规则

```
锁定条件：连续5次登录失败
解锁方式：
  1. 自动解锁：10分钟后自动解除锁定
  2. 管理员手动解锁（后续扩展）
```

### 4.4. 会话规则

```
会话有效期：2小时（从登录成功时刻开始计算）
超时处理：自动跳转到登录页面，提示"会话已过期，请重新登录"
```

## 5. 接口说明

### 5.1. 登录接口

| 属性 | 值 |
|------|-----|
| **接口路径** | `/api/users/login` |
| **HTTP方法** | POST |
| **Content-Type** | application/json |

#### 请求体

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户名（4-50字符） |
| password | String | 是 | 密码（6-32字符） |

#### 成功响应（200 OK）

```json
{
  "message": "登录成功",
  "data": {
    "userId": 1,
    "username": "admin",
    "name": "管理员",
    "role": "admin",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

#### 失败响应（401 Unauthorized）

```json
{
  "message": "用户名或密码错误"
}
```

#### 锁定响应（403 Forbidden）

```json
{
  "message": "账号已锁定，请10分钟后再试"
}
```

## 6. 前端实现说明

### 6.1. 登录表单元素

| 元素ID | 类型 | 说明 |
|--------|------|------|
| `username` | input[type="text"] | 用户名输入框 |
| `password` | input[type="password"] | 密码输入框 |
| `login` | button | 登录按钮 |

### 6.2. 校验函数说明

```javascript
/**
 * 登录表单校验函数
 * @returns {boolean} - 校验是否通过
 */
function validateLogin() {
    // 1. 用户名非空校验
    // 2. 用户名长度校验（4-50字符）
    // 3. 用户名格式校验（仅支持中文、英文、数字、下划线）
    // 4. 密码非空校验
    // 5. 密码长度校验（6-32字符）
    // 返回：校验通过返回true，失败返回false并显示错误提示
}
```

### 6.3. 登录流程

```
用户输入 → 点击登录按钮
    ↓
调用 validateLogin() 进行前端校验
    ↓
校验失败 → 显示错误提示，停留在登录页
    ↓
校验通过 → 发送POST请求到 /api/users/login
    ↓
后端验证成功 → 跳转到工作台页面
    ↓
后端验证失败 → 显示错误提示，失败计数+1
    ↓
失败达到5次 → 账号锁定，显示锁定提示
```
