# 文件浏览器文件夹排序规则

## 排序原则

1. **按功能分类**: 相同类型的文件放同一目录
2. **按时间顺序**: 同类型文件按日期排序
3. **按版本管理**: 不同版本的文件明确标注

## 推荐目录结构

```
测试全流程框架/
├── 01_requirements 需求管理/     # 需求文档
├── 02_data_数据管理/            # 测试数据
├── 03_environments_环境配置/    # 环境配置
├── 04_suites_测试套件/          # 测试套件
├── 05_utils_工具库/             # 工具函数
├── 06_pages_页面对象/           # POM页面对象
├── 06_controllers_控制器/       # 控制器
├── 07_testcases_测试用例/        # 测试用例
├── 08_tests_自动化测试/          # 自动化测试脚本
├── 09_reports_测试报告/          # 测试报告
├── 10_api_test_接口测试/        # 接口测试
├── 11_config_配置管理/          # 项目配置
├── 12_tools_工具/               # 辅助工具
└── 13_locust_压力测试/          # 压力测试
```

## 文件组织规则

| 文件类型 | 存放目录 | 命名规范 |
|---------|---------|---------|
| 需求文档 | 01_requirements | `{模块}_PRD.docx` |
| 测试用例CSV | 07_testcases/csv_表格用 | `{日期}_{模块}_测试用例.csv` |
| 测试用例JSON | 07_testcases/json | `{日期}_{模块}_测试用例_{版本}.json` |
| 接口文档 | 01_api_docs | `swagger.yaml` |
| 测试报告 | 09_reports | `{日期}_{模块}_报告.html` |
| 页面对象 | 06_pages | `{模块}_page.py` |

## 排序优先级

1. 数字前缀（01, 02, 03...）
2. 功能重要性
3. 工作流程顺序
4. 时间倒序（最新在前）
