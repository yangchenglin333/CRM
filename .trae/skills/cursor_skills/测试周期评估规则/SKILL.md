# 测试周期评估规则

## 评估因素

| 因素 | 说明 | 影响系数 |
|------|------|---------|
| 功能复杂度 | 功能难度 | 1.0-2.0 |
| 测试类型 | 功能/性能/安全 | 1.0-1.5 |
| 用例数量 | 预期用例数 | 线性 |
| 回归范围 | 回归测试比例 | 1.2-3.0 |
| 环境准备 | 测试环境就绪 | 1.0-1.3 |

## 工时估算

### 功能测试
| 用例数量 | 预估工时 | 说明 |
|---------|---------|------|
| 1-50 | 1-3天 | 小模块 |
| 50-200 | 3-7天 | 中等模块 |
| 200-500 | 7-15天 | 大模块 |
| 500+ | 15-30天 | 超大模块 |

### 回归测试
| 回归比例 | 系数 |
|---------|------|
| <20% | 1.2 |
| 20%-50% | 1.5 |
| 50%-80% | 2.0 |
| >80% | 3.0 |

## 测试阶段

| 阶段 | 产出物 | 里程碑 |
|------|-------|--------|
| 测试准备 | 测试计划、用例 | Day 1-2 |
| 用例执行 | 执行记录、缺陷 | Day 3-10 |
| 回归测试 | 回归报告 | Day 11-12 |
| 测试报告 | 测试报告 | Day 13 |

## 计算公式

```
总工时 = 功能测试工时 × 复杂度系数 + 回归测试工时 × 回归系数 + 环境准备工时
```

## 风险因素

1. 需求变更
2. 环境问题
3. 人员变动
4. 阻塞项未解决
5. 缺陷修复延迟
