# 文件命名规则

## 通用规则

1. 使用有意义的名称
2. 使用中文或英文，避免中英混合
3. 使用下划线 `_` 分隔词汇
4. 日期格式：`YYYYMMDD`
5. 版本格式：`v1.0.0` 或 `_1.0.0`

## 文件类型后缀

| 类型 | 后缀 | 示例 |
|------|------|------|
| 测试用例CSV | `_测试用例` | `20260430_CRM_测试用例_登录模块.csv` |
| 测试用例JSON | `_测试用例_{版本}.json` | `20260430_CRM_API测试用例_1.0.0.json` |
| PRD文档 | `_PRD.docx` | `CRM系统_PRD.docx` |
| 需求文档 | `_需求总结.md` | `ERP采购模块_需求总结.md` |
| 流程图 | `_流程图.md` | `CRM客户管理_流程图.md` |
| API测试用例 | `_API测试用例_{版本}` | `20260430_CRM_API测试用例_1.0.0.csv` |

## 时间戳格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `YYYYMMDD` | 8位日期 | `20260430` |
| `YYYYMMDD_HHMMSS` | 精确到秒 | `20260430_143052` |
| `YYYYMMDD_HHMMSS_nnn` | 精确到毫秒 | `20260430_143052_123` |

## 项目文件命名

### 测试框架文件
```
{日期}_{项目名}_{文件类型}_{版本}
```

### 报告文件
```
{日期}_{项目名}_{报告类型}_{版本号}
```

## 目录命名

- 使用中文命名
- 清晰表达目录用途
- 避免过长名称
