# 需求文档生成规则

## 文档结构

### 1. 产品概述
- **1.1 背景**: 为什么做这个产品
- **1.2 目标**: 要达成什么目标
- **1.3 范围**: 包含什么，不包含什么
- **1.4 术语表**: 专业术语解释

### 2. 用户分析
- **2.1 目标用户**: 谁来使用
- **2.2 用户角色**: 角色划分
- **2.3 用户画像**: 典型用户描述

### 3. 功能需求
- **3.1 功能模块划分**: 模块结构
- **3.2 功能详细描述**: 每个功能的详细说明

### 4. 非功能需求
- **4.1 性能要求**: 响应时间、并发数
- **4.2 安全要求**: 认证、授权、数据安全
- **4.3 兼容性要求**: 浏览器、设备支持

### 5. 业务流程
- **5.1 业务流程图**: Visio/流程图
- **5.2 流程说明**: 文字描述

### 6. 接口需求
- **6.1 API列表**: 所有接口
- **6.2 接口详情**: 参数、返回值

### 7. 数据字典
- **7.1 数据表结构**: 数据库设计
- **7.2 字段说明**: 字段含义

## 格式要求

- 使用 Markdown 格式
- 标题层级清晰
- 表格展示结构化信息
- 代码块展示示例

## 输出位置

```
测试全流程框架/01_requirements 需求管理/
├── PRD_{项目名}.docx
└── docs_需求文档/
    └── {模块}_需求总结.md
```

## 文档命名

```
{项目名}_{文档类型}_{版本}
示例：CRM系统_PRD_1.0.0.docx
```

## 评审流程

1. 草案编写 → 2. 内部评审 → 3. 修改完善 → 4. 正式发布
