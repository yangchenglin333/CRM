# 需求提炼规则

## 提炼流程

### Step 1: 原始需求分析
- 阅读原始需求文档
- 识别关键信息点
- 标记不确定内容

### Step 2: 功能点提取
- 列出所有功能点
- 按模块分类
- 确定功能优先级

### Step 3: 需求结构化
- 整理为标准格式
- 补充遗漏信息
- 验证完整性

## 需求结构

### 用户故事格式
```
作为 [用户角色]
我想要 [功能]
以便 [实现价值]
```

### 验收标准格式
```
假设 [前置条件]
当 [操作]
那么 [预期结果]
```

## 功能点描述要素

| 要素 | 说明 | 必填 |
|------|------|------|
| 功能名称 | 简洁明了 | 是 |
| 功能描述 | 详细说明 | 是 |
| 用户角色 | 谁使用 | 是 |
| 前置条件 | 执行前提 | 是 |
| 操作步骤 | 如何执行 | 是 |
| 预期结果 | 预期输出 | 是 |
| 后置条件 | 执行后状态 | 否 |
| 优先级 | 重要程度 | 是 |

## 需求评审要点

1. 需求完整性
2. 需求一致性
3. 需求可测试性
4. 需求可行性
5. 需求描述清晰度

## 输出格式

```markdown
## 功能模块

### 功能点1
- **用户故事**: 作为...我想要...
- **验收标准**:
  1. 假设...当...那么...
  2. 假设...当...那么...
- **优先级**: P0
- **备注**: ...
```
