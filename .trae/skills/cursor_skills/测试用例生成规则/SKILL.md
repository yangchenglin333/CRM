# 测试用例生成规则

## 用例格式

### CSV格式（给人看）
| 列名 | 说明 |
|------|------|
| 序号 | 用例编号 |
| 模块 | 功能模块 |
| 用例名称 | 用例名称 |
| 步骤 | 测试步骤 |
| 预期结果 | 预期结果 |
| 优先级 | P0/P1/P2/P3 |
| 备注 | 测试类型 |

### JSON格式（给AI执行）
```json
{
  "case_id": "TC_001",
  "case_name": "用例名称",
  "priority": "P0",
  "module": "模块名称",
  "steps": [...]
}
```

## 测试方法

- 边界值分析
- 等价类划分
- 错误猜测法
- 场景法
- 因果图法

## 覆盖率要求

| 类型 | 覆盖率 |
|------|--------|
| 正向用例 | 100% |
| 反向用例 | ≥80% |
| 边界值 | ≥20% |
