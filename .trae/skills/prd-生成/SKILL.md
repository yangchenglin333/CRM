# PRD 产品需求文档生成器

## 功能说明

根据用户提供的信息（需求描述、讨论内容、功能列表等），自动生成标准化的PRD产品需求文档。

## 应用场景

- 根据需求讨论生成PRD
- 根据功能列表生成PRD
- 补充完善现有PRD
- 生成标准格式的需求文档
- CRM系统产品需求文档生成

## 工作流程

### Step 1: 收集需求信息
1. 分析用户提供的内容（PRD文档、需求描述、讨论记录）
2. 提取功能需求清单
3. 识别非功能需求（性能、安全、兼容性）
4. 整理业务流程和用户故事
5. 提取接口需求和数据字典

### Step 2: 生成PRD结构
标准PRD包含：
- **产品概述**: 产品背景、目标、范围、术语表
- **用户分析**: 目标用户、用户角色、用户画像
- **功能需求**: 模块划分、功能点详细描述（含用户故事、验收标准）
- **非功能需求**: 性能、安全、兼容性、可靠性等
- **业务流程**: 流程图、文字说明
- **接口需求**: API设计、接口详情
- **数据字典**: 数据结构定义
- **原型链接**: 产品原型地址

### Step 3: 输出文档
- Markdown格式（便于编辑）
- Word格式（正式文档）
- HTML格式（在线浏览）

## 优先级定义

| 级别 | 说明 | 标记 |
|------|------|------|
| P0 | 核心功能，必须实现 | Must |
| P1 | 重要功能，优先实现 | Should |
| P2 | 一般功能，尽量实现 | Could |
| P3 | 备选功能，按需实现 | Won't |

## 验收标准要求

每个功能点必须包含：
1. **用户故事**: As a (角色), I want (功能) so that (收益)
2. **前置条件**: 执行前需要满足的条件
3. **操作步骤**: 详细的功能操作步骤
4. **预期结果**: 功能的预期输出
5. **后置条件**: 执行后系统状态
6. **优先级**: P0/P1/P2/P3

## 输出格式

生成的文件格式：
- `.docx` - Word格式（正式文档）
- `.md` - Markdown格式（便于协作）
- `.pdf` - PDF格式（存档）
