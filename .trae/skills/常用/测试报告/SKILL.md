# 测试报告生成器

## 功能说明

统一管理所有类型测试报告的生成，使用**固定样式模板**，确保报告风格一致性。

## ⚠️ 重要声明

**报告样式已固定为标准模板，不允许随意更改。**

如需更改报告样式，必须经过以下流程：
1. 用户提出样式更改需求
2. AI进行第一次确认："您确定要更改报告样式吗？更改后将影响所有类型的测试报告"
3. 用户再次确认要求更改
4. 只有用户两次确认后，才能进行样式更改

## 应用场景

当用户提出以下任一诉求时，**自动应用本规则**：

- 要求生成测试报告
- 说"生成测试报告"
- 说"导出报告"
- 要求查看测试结果
- 执行完测试后要求生成报告

## 报告类型支持

| 报告类型 | 说明 | 样式模板 |
|---------|------|---------|
| UI自动化测试报告 | Playwright/Selenium等 | 固定模板A |
| 接口测试报告 | API测试、REST/GraphQL | 固定模板B |
| 性能测试报告 | Locust/JMeter | 固定模板C |
| 单元测试报告 | pytest/Jest | 固定模板D |
| 综合测试报告 | 混合多种测试类型 | 固定模板E |

## 标准报告模板

### UI自动化测试报告模板（模板A）

#### ⚠️ 强制要求：使用独立模板文件

**模板文件位置（只读）：**
```
/Users/admin/Desktop/test-management-platform 2/.trae/skills/常用/测试报告/templates/UI自动化报告模板.html
```

**生成报告时必须：**
1. 读取上述模板文件的完整内容
2. 替换所有占位符
3. 不得修改模板文件本身
4. 不得在模板外自行编写HTML/CSS样式

**占位符说明：**
| 占位符 | 说明 |
|--------|------|
| `{project_name}` | 项目名称 |
| `{start_time}` | 开始时间 |
| `{end_time}` | 结束时间 |
| `{total}` | 总用例数 |
| `{passed}` | 通过数 |
| `{failed}` | 失败数 |
| `{pass_rate}` | 通过率 |
| `{duration}` | 执行耗时（秒） |
| `{case_rows}` | 表格行HTML |
| `{screenshot_mapping}` | 截图路径JSON对象 |
| `{detail_sections}` | 详细步骤HTML |

#### 报告结构

```
{报告目录}/{YYYYMMDD}-{项目名}测试/
├── test_report.html          # HTML测试报告（主报告）
├── test_report.json          # JSON测试报告
├── test_case_details.csv     # 用例详情
├── screenshots/              # 截图目录
│   └── ...
└── playback_scripts/         # 可复用测试脚本
    └── ...
```

#### HTML报告必须包含的部分

**第一部分：用例执行列表**
- 表格形式：用例ID、用例名称、状态、截图、备注
- 点击"查看截图"在表格下方显示截图
- 使用 `href="javascript:void(0)"` 防止页面跳转

**第二部分：详细执行步骤**
- 每个Case单独展示
- 步骤信息：步骤编号、操作、预期结果、实际结果、状态
- 每个Case底部只有一个最终结果截图
- 失败步骤用红色边框标记

## 报告生成流程

1. **确定报告类型**
   - UI自动化 → 使用模板A
   - 接口测试 → 使用模板B
   - 性能测试 → 使用模板C

2. **创建报告目录**
   - 位置：`{框架目录}/09_reports_测试报告/`
   - 子目录：`{YYYYMMDD}-{项目名}测试/`

3. **读取模板文件**
   - 读取 `/Users/admin/Desktop/test-management-platform 2/.trae/skills/常用/测试报告/templates/UI自动化报告模板.html`

4. **生成报告文件**
   - HTML主报告（替换占位符）
   - JSON数据报告
   - CSV详情报告

5. **填充报告内容**
   - 汇总统计
   - 用例列表
   - 详细步骤
   - 截图引用

## 注意事项

### ⚠️ 强制要求

1. **样式固定**：所有测试报告必须使用本规则定义的固定模板
2. **不可随意更改**：报告样式为标准规范，不得随意修改
3. **两次确认**：如需更改样式，必须获得用户两次明确确认
4. **完整性**：报告必须包含统计概览、用例列表、详细步骤、截图
5. **模板引用**：必须读取独立模板文件，不得复制粘贴模板代码到报告中

### 🚫 禁止事项

- 禁止使用非标准模板生成报告
- 禁止自行决定更改报告样式
- 禁止省略报告必要组成部分
- 禁止使用截图链接 `href="#"`（必须使用 `href="javascript:void(0)"`）
- 禁止在模板外自行编写HTML/CSS样式

4. 更新独立模板文件：`/Users/admin/Desktop/test-management-platform 2/.trae/skills/常用/测试报告/templates/UI自动化报告模板.html`
5. 通知用户模板已更新
